# --privileged=true

如果是CentOS7安全模块会比之前系统版本加强，不安全的会先禁止，所以目录挂载的情况被默认为不安全的行为，在SELinux里面挂载目录被禁止掉了额，如果要开启，我们一般使用--privileged=true命令，扩大容器的权限解决挂载目录没有权限的问题，也即使用该参数，container内的root拥有真正的root权限，否则，container内的root只是外部的一个普通用户权限。

Docker挂载主机目录访问如果出现cannot open directory .: Permission denied 解决办法：在挂载目录后多加一个--privileged=true参数即可

# 读写规则添加

读写权限（默认）

 docker run -it --privileged=true -v /宿主机绝对路径目录:/容器内目录:rw      镜像名

只读权限

容器内部被限制为只能读取不能写入

 docker run -it --privileged=true -v /宿主机绝对路径目录:/容器内目录:ro      镜像名

# 数据卷继承和共享

  1. 容器1完成与宿主机的映射
[root@localhost centosShare]# docker run -it --privileged=true -v /data/centosShare:/data:ro centos /bin/bash
[root@42e83fe848a2 /]# cd /data/
[root@42e83fe848a2 data]# ls
testShare.txt
  1. 容器2继承容器1的数据卷规则
[root@localhost centosShare]# docker run -it --privileged=true --volumes-from u1 --name u2 centos
[root@22b0bf176b47 /]# cd /data/
[root@22b0bf176b47 data]# ls
testShare.txt
