# 镜像命令

列出本地主机的镜像

[root@localhost docker]# docker images
REPOSITORY    TAG       IMAGE ID       CREATED       SIZE
hello-world   latest    9c7a54a9a43c   6 weeks ago   13.3kB

  • REPOSITORY：表示镜像的仓库源

  • TAG：镜像的标签版本号

  • IMAGE ID：镜像ID

  • CREATED：镜像创建时间

  • SIZE：镜像大小

同一REPOSITORY可以有多个TAG，代表不同版本，如果docker run 时不指定版本默认使用的是latest版本。

列出本地所有镜像含历史镜像

docker images -a

只显示镜像ID

docker images -q

从Hub上搜索某个镜像

docker search 镜像名称

删除镜像

# 删除单个
docker rmi -f 某个镜像ID
# 删除多个
docker rmi -f 镜像名1:TAG 镜像名2:TAG
# 删除全部
docker rmi -f $(docker images -qa)

# 容器命令

新建启动容器

docker run [OPTIONS] IMAGE [COMMAND] [ARG...]

OPTIONS：

  • --name="新容器名字"，为容器指定一个名称
  • -d后台运行并且返回容器ID，启动守护式容器（后台运行）
  • -i以交互模式运行容器，通常与-t同时使用
  • -t为容器分配一个伪输入终端，通常与-i同时使用，启动交互式容器（前台伪终端，等待交互）
  • -P随机端口映射
  • -p指定端口
  • --volumes-from 父容器名称，继承父容器的数据卷位置
# 端口映射 -p 8080:80
-p hostPort:containerPort
# 配置监听地址 -p 10.0.0.100:8080:80
-p ip:hostPort:containerPort
# 随机分配端口 -p 10.0.0.100::80
-p ip::containerPort
# 指定协议 -p 8080:80:tcp
-p hostPort:containerPort:udp
# 指定多个
-p 81:80 -p 443:443

以交互模式启动运行centos容器，并且执行/bin/bash命令，退出终端直接输入exit

[root@localhost docker]# docker run -ti centos /bin/bash
[root@20886be28070 /]# ps -ef  
UID         PID   PPID  C STIME TTY          TIME CMD
root          1      0  0 07:18 pts/0    00:00:00 /bin/bash
root         15      1  0 07:18 pts/0    00:00:00 ps -ef
[root@20886be28070 /]# exit
exit

列出正在运行容器

docker ps [OPTIONS]

OPTIONS说明

  • -a列出所有正在运行的容器和历史上运行过的容器。

  • -l显示最近创建的容器。

  • -n 个数显示最近n个创建的容器。

  • -q静默模式，只显示容器编号。

退出容器

  • exit run进去容器，执行exit，容器停止

  • ctrl+p+q run进去容器，ctrl+p+q，容器不停止

启动已停止的容器

docker start 容器ID或者容器名(NAMES)

重启容器

docker restart 容器ID或者容器名

停止容器

docker stop 容器ID或者容器名

强制停止容器

docker kill 容器ID或者容器名

删除已停止的容器

# 删除单个容器
docker rm 容器ID
# 一次性删除多个容器实例
docker rm -f $(docker ps -a -q)
docker ps -a -q | xargs docker rm

为什么守护式启动Centos，容器还是会自动退出？

[root@localhost docker]# docker run -d centos
3ad3659b45c2022da1d4bcd8821ee41ccf08cac34e45a03d5ece4046bb126124
[root@localhost docker]# docker ps -a
CONTAINER ID   IMAGE     COMMAND       CREATED         STATUS                     PORTS     NAMES
3ad3659b45c2   centos    "/bin/bash"   7 seconds ago   Exited (0) 5 seconds ago             cool_satoshi
c30ced2516e6   centos    "/bin/bash"   8 minutes ago   Exited (0) 8 minutes ago             sleepy_agnesi

docker机制：docker后台运行必须有一个前台进程。如果容器运行的命令不是一直挂起的命令就会自动退出。如果某个容器后台模式运行，docker前台没有运行的程序，这样的容器后台启动后，会觉得没有事情可做立即自杀，常见解决方案就是以前台进程形式运行，常见就是命令模式，表示有交互。

# 后台模式运行redis
docker run -d redis:6.0.8
# 交互模式运行redis
docker run -it redis:6.0.8

查看容器日志

docker logs 容器ID/容器名称

容器内运行的进程

docker top 容器ID/容器名称

容器内部细节

docker inspect 容器ID/容器名称

进入正在运行容器并以命令行交互

docker exec -it 容器ID/容器名称 bashshell

docker attach 容器ID/容器名称

  • exec打开新的终端，并且可以启动新的进程，所以用exit退出，不会导致容器停止，推荐使用。
  • attach直接进入容器启动命令的终端，不会启动新的进程，所以用exit退出，会导致容器停止。

从容器内拷贝文件到主机

docker ps 容器ID:容器内文件位置 主机位置

容器停止了也是可以复制的

容器导入和导出

export：导出容器内容作为一个tar文件

docker export 容器ID > 文件名字.tar

import：将tar中内容作为新的文件系统导入为镜像

cat 文件名字.tar | docker import - 用户名/镜像名字:镜像版本号

# 其他命令

  • attach 当前 shell 下 attach 连接指定运行镜像

  • build 通过 Dockerfile 定制镜像

  • commit 提交当前容器为新的镜像

  • cp 从容器中拷贝指定文件或者目录到宿主机中

  • create 创建一个新的容器，同 run，但不启动容器

  • diff 查看 docker 容器变化

  • events 从 docker 服务获取容器实时事件

  • exec 在已存在的容器上运行命令

  • export 导出容器的内容流作为一个 tar 归档文件[对应 import ]

  • history 展示一个镜像形成历史

  • images 列出系统当前镜像

  • import 从tar包中的内容创建一个新的文件系统映像[对应export]

  • info 显示系统相关信息

  • inspect 查看容器详细信息

  • kill kill 指定 docker 容器

  • load 从一个 tar 包中加载一个镜像[对应 save]

  • login 注册或者登陆一个 docker 源服务器

  • logout 从当前 Docker registry 退出

  • logs 输出当前容器日志信息

  • port 查看映射端口对应的容器内部源端口

  • pause 暂停容器

  • ps 列出容器列表

  • pull 从docker镜像源服务器拉取指定镜像或者库镜像

  • push 推送指定镜像或者库镜像至docker源服务器

  • restart 重启运行的容器

  • rm 移除一个或者多个容器

  • rmi 移除一个或多个镜像[无容器使用该镜像才可删除，否则需删除相关容器才可继续或 -f 强制删除]

  • run 创建一个新的容器并运行一个命令

  • save 保存一个镜像为一个 tar 包[对应 load]

  • search 在 docker hub 中搜索镜像

  • start 启动容器

  • stop 停止容器

  • tag 给源中镜像打标签

  • top 查看容器中运行的进程信息

  • unpause 取消暂停容器

  • version 查看 docker 版本号

  • wait 截取容器停止时的退出状态值
