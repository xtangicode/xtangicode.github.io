通常我们使用缓存如果没有特别复杂的场景，使用ConcurrentHashMap的对象就可以实现。但是如果有失效时间、缓存大小等限制可以采用缓存框架，常用的缓存框架EhCache、Caffeine、GuavaCache。

# 集成Caffeine

# 引入maven依赖

<dependency>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-cache</artifactId>
</dependency>
<dependency>
  <groupId>com.github.ben-manes.caffeine</groupId>
  <artifactId>caffeine</artifactId>
  <version>2.6.0</version>
</dependency>

# 创建全局配置类

package com.sugar.framework.cache;


import com.github.benmanes.caffeine.cache.Cache;
import com.github.benmanes.caffeine.cache.Caffeine;
import com.google.common.collect.Lists;
import org.springframework.cache.CacheManager;
import org.springframework.cache.caffeine.CaffeineCache;
import org.springframework.cache.caffeine.CaffeineCacheManager;
import org.springframework.cache.support.SimpleCacheManager;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.context.annotation.Primary;

import java.util.ArrayList;
import java.util.Map;
import java.util.concurrent.ConcurrentHashMap;
import java.util.concurrent.TimeUnit;

/**
 * @description:
 * @author: Tx
 * @date: 2022-12-21  13:24
 */
@Configuration
public class CacheConfig {

    @Bean("CacheManager")
    @Primary
    public CacheManager cacheManager () {
        SimpleCacheManager cacheManager = new SimpleCacheManager();
        ArrayList<CaffeineCache> caches = new ArrayList<CaffeineCache>();
            caches.add(new CaffeineCache("view",
                Caffeine.newBuilder().recordStats()
                .expireAfterWrite(24, TimeUnit.HOURS)
                .maximumSize(100)
                .build())
        );
        cacheManager.setCaches(caches);
        return cacheManager;
    }

}

# 配置注解

在方法上添加@Cacheable注解，指定使用的缓存的名称value和缓存中的key，第一次访问成功存放到缓存。后续访问直接从缓存中取，直到缓存失效重新查询放到缓存中。

@Cacheable(value = "view", key = "#param.key")
public Object getVO(PortQuery param) {
    //....
}

使用场景

  • 需要BI数据，需要大量图表查询或者经常访问，且对数据强一致性实时性要求不高。
  • 查询缓慢，且数据刷新频率不高的数据。

方案

  • 整合定时任务**@Scheduled**、@EnableScheduling，定时刷新缓存防止缓存过期并且更新缓存中数据。由于缓存可设置写入缓存后多长时间失效。
  • 如果查询缓慢可以在项目启动时，先进行缓存预热，一般可以在**@PostConstruct**修饰的方法中进行预热缓存。
